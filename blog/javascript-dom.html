JavaScript Fundamentals

What's an analogy to describe JavaScript and its relationship to HTML and CSS?


Think of HTML, CSS and Javascript as a dynamic trio. HTML contains the structure and content of a page, CSS is the misunderstood artiste who makes the HTML content look pretty, and JavaScript is the wizard that brings everything to life on the page (like interactivity and effects).


What are 'control flow' and 'loops'?


Let’s use the cute example of gardening to illustrate this.


Control flow: refers to the order in which actions or steps are executed in a process. In gardening, this could be making decisions about the order of jobs to carry out throughout the year.


Loops: are repetitive actions or steps performed until a specific condition is met. In this case, loops will be the maintenance tasks for ongoing garden care.

  1. Planning the garden (control flow): You need to decide what to grow and location for planting.
  2. Preparing the soil (control flow): After planning, you prepare the soil (shout out to Charles Dowding’s ‘no dig’ methods - if you aren’t already a fan of his YouTube content, prepare yourself for a wholesome dive into the world of ‘no dig’).
  3. Planting seeds (control flow): You plant seeds or seedlings according to your garden plan.
  4. Watering (control flow): You decide how much watering is required depending on the specific crops and weather conditions.
  5. Ongoing watering (loops): Watering is an ongoing task that occurs in loops. This process is repeated depending on whether certain soil moisture ‘conditions’ are met.
  6. Weeding (control flow): Once again, this is where Charles Dowding’s golden advice swoops in and you decide to cover weeds with cardboard or tarp to manage the majority of them.
  7. Ongoing weeding (loops): You periodically inspect whether the odd weed pops through, and you can remove them as they appear.
  8. Fertilising (control flow): You decide whether to use fertilisers or not, and apply them as needed.
  9. Ongoing care (loops): You regularly prune, fertilise, monitor for pests and diseases entering loops for each of these repetitive tasks.
  10. Harvesting (control flow): When the plants are ready, you harvest the fruit and vegetables.
  11. End of season clear up and plan for the next season (control flow): You clear any produce from the garden and prepare it for the next season.

What is the DOM?


The Document Object Model (DOM) is not a physical or visible part of a web page; rather, it's a programming interface provided by web browsers that represents the structure and content of a web page as a hierarchical tree-like structure. At the top of the DOM tree is a single object called the "document" object. This represents the entire web page. From there, the tree branches out to include elements and their child elements, forming a hierarchy.


The DOM is created by the browser when it loads a web page, and it exists in memory as an abstract representation of the page's structure and content. When you press F12 or use your browser's developer tools to inspect a web page, what you're seeing is an interface that allows you to interact with and inspect the Document Object Model (DOM) of that web page. So, in a way, you are viewing the DOM, but it's presented to you in a more user-friendly and interactive format.


Here's an analogy to illustrate the DOM:


Think of a web page as a book with different chapters and pages. The DOM is like an index or table of contents for that book, providing a structured way to access and modify each chapter or page.


Example of interacting with the DOM


Let's say you have a web page with a button on it, and you want to change the text displayed on that button when a user clicks it. Here's how you might interact with the DOM in JavaScript to achieve this:




  1. There is a button with the id 'myButton'.

  2. In the "script" element, you use the JavaScript code 'document.getElementById('myButton')' to access and store a reference to this button element in the DOM.

  3. We add an event listener to the button to listen for the 'click' event.

  4. When the button is clicked, the event listener function is triggered and it changes the button's text using 'button.text.Content'.

What's the difference between accessing data from arrays and objects?


Arrays are like a line of people holding hands, and objects are like a set of drawers with labelled handles.


Arrays: Imagine a line of people standing in a row, each person holding the hand of the person next to them. To reach a specific person in the line, you count from the beginning or end of the line to find the person you're looking for. For example, if you want to find the 5th person, you count 1, 2, 3, 4, 5, and then you're there. In arrays, data is accessed by position or index, much like finding a person in a human chain.


Objects: Imagine a pantry with various containers, each labelled with a different category like "Cereals," "Spices," or "Snacks." Inside each container, you have items related to that category. To find a particular item, you don't need to know its position; you simply look for the container with the right label, open it, and grab what you need. In objects, data is organised by labels or keys, much like how items in a pantry are sorted into labelled containers.


Depending on your needs, you might choose one over the other for organising and accessing your data



What are functions and why are they helpful?


Functions are like ‘containers’ to store information or instructions. They are helpful because they make your code organised, reusable and you can ‘call’ functions to perform certain things. Imagine a function like a recipe card:


  1. Every recipe card (function) has a name, and when you say its name (or ‘call the function’), JavaScript gets cooking.
  2. Think of the ingredients in the recipe card as the ‘parameters’ of the function.
  3. The recipe instructions are the ‘code’ inside the function. This tells JavaScript what to do with the ingredients to make the recipe.
  4. You can reuse the recipe card over and over again by calling it. You don’t need to write out the instructions every time.
  5. After the recipe is made, JavaScript can give you back the meal as a result, or a ‘return’.
